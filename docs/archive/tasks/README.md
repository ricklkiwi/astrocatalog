# Archived Task Documents

Completed implementation task plans and specs live here.

Each completed task gets one folder:

```text
<slug>/
  plan.md
  spec.md
```

The orchestrator archives task docs after reviewer PASS and before opening the PR. It must also update repo links so `docs/plans/<slug>.md` and `docs/specs/<slug>.md` do not remain as stale references.
